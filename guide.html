🧠 The Decision Tree
Ask yourself these questions for every person.
"Is this a real person or just a name?"
If you only have a name and they haven't signed up or paid...
⏳ CREATE PHANTOM → Onboard tab → type name → "+ Add as Pending"
"Did they pay?"
They're a real person ready to join. They paid or they want to build for free.
YES → Submit request on My Codes → Keith approves → ⚡ Onboard
NO but wants to build → Onboard as "Affiliate Only — Unpaid" (no code needed)
"Do we know where they go in the tree?"
If you know their upline, place them directly. If not, hold them.
YES → Select the person in the 🔍 Referrer dropdown → direct placement
NO → Select "📦 Holding Tank" → Keith places them later
"Their upline hasn't signed up yet?"
The person they go under isn't in the system yet.
Create the upline as PHANTOM first → then select that phantom in the dropdown
💎 Flow: Onboarding a Paid Member
The most common flow. Someone paid, you onboard them.
My Codes tab → fill in name, email, package, VPN, payment status → Submit to SBA HQ
Button animates → green success card → appears in ⏳ Waiting queue
Keith approves → codes push to your wallet (📦 package + 🔒 VPN)
Cards appear at top of My Codes with info summary
Click ⚡ Onboard → form auto-fills name, email, codes, package, VPN, payment
Only thing left: pick the referrer
🔍 Search the referrer dropdown → find the right person → select them
Or Holding Tank if unsure. Or create a phantom if upline isn't here yet.
Click ⚡ Onboard Member → system creates account, burns codes, calculates commissions
Welcome email popup → click 📋 Copy or 📨 Auto-Send → member gets login + Discord link + instructions
🆓 Flow: Free Affiliate (No Package)
They want to build a team but haven't bought yet.
Go to Onboard tab directly (no code request needed)
Fill in name and email
Member Type: Affiliate Only — Unpaid
No activation code needed — leave it blank
Pick referrer (or Holding Tank) → Onboard
They get a portal login, referral link, AI Closer — but no Genius betting tools
⏳ Flow: Creating Phantoms
Placeholder names for people who haven't signed up yet.
Single phantom: Onboard tab → type name in "Upline not in system yet?" field → + Add as Pending
Batch phantoms: Playbook tab → Batch Add Phantoms → type names (one per line) → Create All as Pending
Phantoms appear in the referrer dropdown as ⏳ Name (pending)
Place real people under phantoms using the dropdown
When the phantom person pays → submit request → Keith approves → onboard them → phantom becomes real, tree stays intact
